💡 [ADK 2.0 Graph Workflow HITL(Human-In-The-Loop) 핵심 설계 분석 및 테스트 검증 요약]
================================================================================

1. **HITL Resume 핵심 메커니즘**:
   - Graph Workflow 상에서 `RequestInput` 노드를 통해 일시 정지(Paused) 상태가 된 노드를 다시 정상적으로 재개(Resume)하려면 단순 신규 독립 메시지(`message`)를 전달하는 것이 아닙니다.
   - 이전 실행 노드로부터 반환받은 고유 ID인 **`interruptId`**를 추적하여, 다음 REST API 요청의 `resume_inputs` 파라미터 맵 하위에 `{"[interruptId]": "새로운 사용자 입력값"}` 형태로 명시해 보내야만 세션 상태가 동기화되며 다음 노드로 완벽히 재개(Resume)가 진행됩니다.

2. **미반영 시 부작용 (Rerun 루프)**:
   - 해당 `interruptId` 및 `resume_inputs` 매핑을 누락한 채 일반 메시지만을 전송하면, 에이전트 런타임은 대화 세션의 이전 히스토리를 이어받지 못하고 매 요청마다 세션을 새로 시작(Rerun)하여 첫 번째 안내 노드에 영원히 머무르는 오작동이 유발됩니다.

3. **테스트 스크립트(`test_workflow.py`) 개선 결과**:
   - 실시간으로 전달되는 SSE(Server-Sent Events) 스트림 결과 로그로부터 동적으로 `interruptId` 및 `response_schema`의 데이터 타입을 파싱하고 자동 추출하는 엔진을 구현하였습니다.
   - 추출된 동적 `interruptId` 값을 자동으로 다음 Multi-turn API 요청의 `resume_inputs` 구조적 페이로드에 결합하여 REST API를 연쇄 전달함으로써, Case 1 ~ Case 3까지의 여행 일정 수립, 피드백 반영, 변경 데이터 가공 루프까지 완벽하게 자동 검증을 통과하도록 리팩토링 및 시나리오 테스트를 고도화했습니다.
